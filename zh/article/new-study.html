js中try finally 执行

js中finally内容执行

    function foo(){
        try{
            return 0;
        } catch(err) {

        } finally {
            console.log("a")
        }
    }

    console.log(foo());
    // a 0  return 执行，但是函数没有立即返回又执行了finally内容 
function foo(){
  try{
    return 0;
  } catch(err) {

  } finally {
    return 1;
  }
}

console.log(foo());

在finally中加return，会覆盖try当中的return 这一套怪异的机制，正式 js语句执行的完成状态，标准类型表示为:Completion Record (描述异常、跳出等语句执行过程)

Completion Record 表示一个语句执行完之后的结果： 1.[[type]] 完成的类型 break continue return throw normal 2.[[value]] 语句的返回值 没有语句则是empty 3.[[target]] 语句的目标 通常是js标签

    • var声明

普通语句执行后，[[type]]为normal的completion record，js 引擎会执行下一条语句

当语句为表达式语句，会产生[[value]]，当然可以在控制器 var 会产生undefined，而不加var 就会生成value

Chrome 控制台显示的正是语句的 Completion Record 的[[value]]

语句块(大括号括起来的一组语句)

需要注意当语句块的Completion Record 的[[type]] 不为normal ，会打断语句块后续的语句执行

{ var i = 1; // normal, empty, empty i ++; // normal, 1, empty console.log(i) //normal, undefined, empty } // normal, undefined, empty

可以看到语句块并没有被打断，即normal

加入一个return，产生了非normal，整个语句变为非normal，也证明了非normal完成类型可以穿透复杂的语句嵌套结构，产生控制效果

{ var i = 1; // normal, empty, empty return i; // return, 1, empty i ++; console.log(i) } // return, 1, empty

控制型语句

  • 对内部造成影响 if 、switch 、 while/for 、 try
  • 对外部造成影响 break 、continue 、return 、 throw

控制语句和break 、continue 、return 、 throw 四种类型 两两组合

因为finally中的内容必须保证执行，所以 try/catch 执行完毕，即使得到的结果是非 normal 型的完成状态，也要执行finally

而finally执行得到非normal 记录，则会使finally中的记录作为整个 try 结构结果

带标签的语句

target字段，JavaScript 中的一个语法，带标签的语句

    //标签语句
    firstStatement: var i = 1;

唯一有作用的是:与完成状态中的target配合，跳出多层循环

    outer:while(true){
        inner:while(true){
            break outer
        }
    }
    console.log('overScanf')

break/continue后跟了关键字,会产生target的完成记录，一旦记录带了target，对应的关键字语句会消费它

js语句存在不同的嵌套关系，在执行过程实际是一个树形结构上执行，它为每个节点执行后产生ComPletion Record ,根据语句的结构和完成记录，js实现了各种分支和跳出逻辑

var var = 1
function foo(){
    console.log(a)

        var a = 2
    }

if(false)不会执行，但是预处理阶段,var会穿透语句结构，它只认脚本/模块/函数体

var a = 1;

function foo() {
    var o= {a:3}
    with(o) {
        var a = 2;
    }
    console.log(o.a);
    console.log(a);
}

foo();

预处理只认var中声明变量，所以foo作用域创建了a，但没有赋值，而with语句会判断有o的情况，加入了作用域a = 2 而执行阶段 到var a = 2(作用域为with 语句内) 所以最终是2 undefined
